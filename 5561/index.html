Real Time Post Reconstruction

Group Members

  • Liam Tyler
  • Bridger Herman
  • Nikki Kyllonen

Description

We implemented a basic real time object tracking system, which you might see extended for motion capture suits in movies, or CAVE VR environements. We setup two infrared cameras on a rig, pointing in the same direction. For objects we wanted to track, we attached three retroreflective markers to them. Every frame that the cameras would take a picture, we would use epipolar geometry to reconstruct the position of the markers, which then was used to determine the rotation of the object. We fed this data into a small renderer, which would move around a model to have the same position and rotation as the markers.

Difficulties

Some of the main difficulties we had were calibrating the cameras. We used the matlab stereo calibration method with a checkerboard image, but there was a lot of difficulty with the OptiTrack cameras and lots of manual effort for the calibration. Another challenge we had was with the OptiTrack API. It turned out to have several limitiations, such as being incapable of transmitting more than one grayscale image at a time, which is how we wanted to originally get the images to perform marker segmentation on.

Features

  • Position reconstruction within one inch
  • Limited rotation reconstruction about the X and Y axis, no rotation in the Z axis (too few degrees of freedom)
  • Real time implementation
  • Smoothing the noise from position reconstruction

Tools Used

  • OpenGL for the rendering
  • SDL2 for the window management and user interaction
  • GLM for the vectors and matrices
  • OptiTrack infrared cameras and API for capturing the images
  • Retroreflective markers for tracking
