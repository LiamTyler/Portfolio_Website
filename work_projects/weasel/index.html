Regression Testing Framework

Description

Vital takes the results of scans such as MRI and CT scans, and builds 3D volumes that users can view and interact with. For the past 20 years though, they had never had a good and consistent way to test the volume building. Developers would continually create new tools when new features / versions came out. This framework that my mentor and I built allows developers to embed what is essentially a logger into their code. They specify the results that are important to record, and then the logger would record this, and send it to a backend, compare it against previous versions of the code, and generate a webpage displaying the results of test. It was very hassle free for the users, and was generic enough to be used on many existing projects, and hopefully many future ones. We made heavy use of c++ templates and flatbuffers to log the results, and get their values on a backend later for comparison.

Tools Used

  • Heavy use of c++ templates
  • Flatbuffers for data serialization
  • Docker for running the backend
  • CMake for compilation and building on Windows, Linux, and Mac

Features

  • Templated API that is very user friendly
  • Serialization and posting to a backend
  • Cross platform compatibility
