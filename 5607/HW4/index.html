Real Time OpenGL

Description

The goal of this assignment was to learn how to do real time 3D rendering with OpenGL. The result was a small dungeon crawler-like game, where you move your character through the dungeon, pick up keys, and open doors to get to the final room and win. See the features and pictures below for more.

Difficulties

The hardest part was just figuring out how to do things in OpenGL. The actual game logic and parsing the map were both simple. For the shaders, I have written Blinn-Phong shaders before, so that wasn't too bad. It was just figuring out how to store everything and interact with OpenGL, and debug mysterious segmentation faults.

Tools Used

  • OpenGL for all of the rendering.
  • SDL2 for the window management, as well as for the keyboard and mouse interaction.
  • Blender to create the key models.

Features

  • Camera follows the character's position to give a first person perspective.
  • Move character with W,A,S,D, and turn head with the mouse.
  • Texturing the walls and doors.
  • Blinn-Phong shading.
  • OBJ model loading and rendering (the keys).
  • Supports variable number of point lights.
  • Player can grab the keys (with 'e') and open the matching colored doors.
  • Door opening is animated.
  • When player reaches final room, screen fades to white to signal win.
